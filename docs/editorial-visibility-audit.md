# Editorial-data visibility audit

Status: ORA-F03 finding, 2026-07-20. This is an inventory of current behavior, not a
governance decision. The choices that require ratification are tracked in
`ORATLAS_DECISIONS.md` §9.

## Boundary used by this audit

A **public boundary** is a response or page that an anonymous caller can retrieve. An
authenticated submission, editorial, moderation, notification, monitoring, or federation
inbox mutation is not a public serializer merely because it returns JSON to its authorized
caller. Those routes still require authorization and request-integrity review, but they do
not determine what belongs in the public scholarly archive.

The following values are unequivocally private under the capability model and are covered by
sentinels: `InspectionCapture.tokenHash` and `InspectionCapture.payloadJson`. Public provenance
may contain deliberately bounded derivatives such as a capture payload hash, submission id,
submitter login, editor login, and publication timestamp. A hash is not permission to return
its source bytes. Submission drafts, diagnostics, notes and decision rationales are inventoried
below but not assigned a permanent visibility by this audit.

### Formal-challenge field matrix

This matrix records the implemented boundary. “Undecided” means the current projection is
deliberately preserved by this audit but still needs the audience decision in §9.

| Challenge datum                                        | Anonymous challenge API/page                       | Editorial queue                                                                                 | Scholarly exports / review JSON-LD | Finding                                                                             |
| ------------------------------------------------------ | -------------------------------------------------- | ----------------------------------------------------------------------------------------------- | ---------------------------------- | ----------------------------------------------------------------------------------- |
| Exact subject, grounds, status, filed hash, timestamps | Included for all five lifecycle states             | Active states only                                                                              | Absent                             | Public status/grounds remain undecided.                                             |
| Challenger login/display name                          | Included                                           | Not included by the queue projection                                                            | Absent                             | Current public attribution remains undecided.                                       |
| Visible challenge body                                 | Included and rendered as escaped text              | Not included by the queue projection                                                            | Absent                             | Current public evidence remains undecided.                                          |
| Contributor response body/hash/login/display name      | Included                                           | Included for `author-responded` rows                                                            | Absent                             | Current public response and attribution remain undecided.                           |
| Transition actor login                                 | Included; this exposes the terminal resolver login | Not separately included                                                                         | Absent                             | Resolver/adjudicator identity is explicitly undecided.                              |
| Transition actor role snapshot                         | Excluded                                           | Excluded                                                                                        | Absent                             | Retained only; audience is undecided.                                               |
| Resolution/dismissal rationale                         | Excluded                                           | Entered and retained through the editor-only workflow, but not returned by the queue projection | Absent                             | Current editor-only projection is undecided.                                        |
| Removed challenge/response body                        | Empty-string tombstone only                        | Empty-string tombstone for a removed response; challenge body is not in this projection         | Absent                             | Retained bytes never cross these serializers; retention/takedown remains undecided. |
| Remover id, role snapshot, removal timestamp           | Excluded                                           | Excluded                                                                                        | Absent                             | Retained only; audience is undecided.                                               |
| Moderation/lifecycle audit event details               | Excluded                                           | Available only through the separately authorized editorial audit surface                        | Absent                             | Audit audience and retention remain undecided.                                      |

The editorial challenge queue itself is not anonymous: `/editorial` redirects signed-out users
and rejects signed-in non-editors before `listOpenChallengePage` is called. Its deterministic,
bounded projection contains only active (`open` and `author-responded`) records and immutable
links. Terminal records leave the queue but remain in the public version-scoped register.

## Findings and closure state

1. No public serializer returns an inspection capability token or raw capture payload.
2. The broad public delivery paths use explicit DTO construction or strict contracts; no route
   returns a raw `Submission`, `InspectionCapture`, notification, audit, or agent-run row.
3. Current behavior deliberately publishes several editorial facts: accepted override
   rationales, open-review reports/responses/decision letters, lifecycle reasons, edge-proposal
   rationales, citation-update resolution notes, replication withdrawal reasons, challenge and
   response text/attribution, and challenge transition actor logins (including terminal resolver
   identity).
4. Those facts are not changed by ORA-F03. Declaring them permanently public—or narrowing them—
   is governance work, so full backlog closure is blocked on `ORATLAS_DECISIONS.md` §9.
5. The bounded sentinel regressions cover the review DTO/API, preservation/PROV export context,
   immutable version diff, archive knowledge index, complete challenge lifecycle, tombstones,
   editorial queue, and challenge exclusion from scholarly exports. Query-shape and contract tests
   remain the proportional guards for independent node, graph, synthesis, feed, federation,
   protocol, and replication serializers.
