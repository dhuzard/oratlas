# Editorial-data visibility audit

Status: ORA-F03 finding, 2026-07-20. This is an inventory of current behavior, not a
governance decision. The choices that require ratification are tracked in
`ORATLAS_DECISIONS.md` §9.

## Boundary used by this audit

A **public boundary** is a response or page that an anonymous caller can retrieve. An
authenticated submission, editorial, moderation, notification, monitoring, or federation
inbox mutation is not a public serializer merely because it returns JSON to its authorized
caller. Those routes still require authorization and request-integrity review, but they do
not determine what belongs in the public scholarly archive.

The following values are unequivocally private under the capability model and are covered by
sentinels: `InspectionCapture.tokenHash` and `InspectionCapture.payloadJson`. Public provenance
may contain deliberately bounded derivatives such as a capture payload hash, submission id,
submitter login, editor login, and publication timestamp. A hash is not permission to return
its source bytes. Submission drafts, diagnostics, notes and decision rationales are inventoried
below but not assigned a permanent visibility by this audit.

## Public pages, embedded data, and exports

| Surface                                    | Finding                                                                                                                                                                                             |
| ------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Review and immutable-version pages         | Render `getReviewDetail`; the review page separately publishes current open-review process history (reports, author responses and decision letters). That current convention needs §9 ratification. |
| Review JSON-LD and export links            | JSON-LD is assembled from the bounded review/export DTOs. No raw Prisma submission or capture record is embedded.                                                                                   |
| Node and node-version pages                | Render public node contracts; node JSON-LD is assembled from the same contract.                                                                                                                     |
| Archive, claims, graph and synthesis pages | Render the public APIs/helpers listed above. They do not receive editorial queue records.                                                                                                           |
| Editorial and round pages                  | Authenticated operational surfaces. They intentionally show workflow data to eligible users and are not public archive boundaries.                                                                  |
| Preserved-file download                    | Returns accepted repository bytes from `RepositorySnapshot.preservedFilesJson`; never the expiring inspection-capability payload.                                                                   |

## Findings and closure state

1. No public serializer returns an inspection capability token or raw capture payload.
2. The broad public delivery paths use explicit DTO construction or strict contracts; no route
   returns a raw `Submission`, `InspectionCapture`, notification, audit, or agent-run row.
3. Current behavior deliberately publishes several editorial facts: accepted override
   rationales, open-review reports/responses/decision letters, lifecycle reasons, edge-proposal
   rationales, citation-update resolution notes, and replication withdrawal reasons.
4. Those facts are not changed by ORA-F03. Declaring them permanently public—or narrowing them—
   is governance work, so full backlog closure is blocked on `ORATLAS_DECISIONS.md` §9.
5. The bounded sentinel regression covers the review DTO/API, preservation/PROV export context,
   immutable version diff, and archive knowledge index. Query-shape and contract tests remain the
   proportional guards for independent node, graph, synthesis, feed, federation, protocol, and
   replication serializers.
